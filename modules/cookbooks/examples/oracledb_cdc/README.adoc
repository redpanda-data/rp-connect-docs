= Oracle Database CDC Examples

Example pipelines for capturing change data from Oracle Database using LogMiner.

== Quick start

. Start the test environment:
+
[,bash]
----
docker compose up -d oracle
----

. Create test data:
+
[,bash]
----
./setup-test-data.sh
----

. Run an example:
+
[,bash]
----
ORACLE_CONNECTION_STRING=oracle://cdc_user:CdcPassword123@localhost:1521/FREEPDB1 \
  rpk connect run basic-capture.yaml
----

. Make changes to see CDC in action:
+
[,bash]
----
docker exec -it oracle-cdc-test sqlplus my_schema/MySchemaPassword123@localhost:1521/FREEPDB1 <<EOF
INSERT INTO ORDERS (ID, CUSTOMER_NAME, TOTAL, STATUS) VALUES (4, 'David', 199.99, 'pending');
UPDATE ORDERS SET STATUS = 'shipped' WHERE ID = 1;
COMMIT;
EXIT;
EOF
----

== Examples

[cols="1,2"]
|===
|File |Description

|`basic-capture.yaml`
|Capture CDC events and output with metadata

|`filter-events.yaml`
|Filter out DELETE events and add operation metadata

|`stream-snapshot.yaml`
|Capture both initial snapshot and ongoing changes

|`to-redpanda.yaml`
|Route CDC events to Redpanda with composite keys

|`to-s3.yaml`
|Archive CDC events to S3 with time partitioning

|`route-by-event.yaml`
|Route different event types to different topics
|===

== Test services

The `docker-compose.yaml` includes optional services for testing outputs:

* *oracle* (port 1521) - Oracle Database Free Edition for CDC input
* *redpanda* (port 9092) - Kafka-compatible broker for Redpanda outputs
* *localstack* (port 4566) - S3-compatible storage for S3 outputs

Start all services:

[,bash]
----
docker compose up -d
----

Start only Oracle:

[,bash]
----
docker compose up -d oracle
----

== Run unit tests

The YAML files include embedded tests that validate the transformation logic:

[,bash]
----
rpk connect test *.yaml
----

This tests the processor logic without requiring a running Oracle instance.

== Environment variables

The examples use these environment variables:

[,bash]
----
export ORACLE_CONNECTION_STRING=oracle://cdc_user:CdcPassword123@localhost:1521/FREEPDB1
export REDPANDA_BROKERS=localhost:9092
export S3_BUCKET=test-bucket
----

== Oracle CDC prerequisites

The `setup-test-data.sh` script configures:

* Supplemental logging at database and table level
* CDC user with LogMiner permissions
* `rpcn` schema for checkpoint storage
* Test tables with sample data

To manually connect to Oracle:

[,bash]
----
docker exec -it oracle-cdc-test sqlplus system/TestPassword123@localhost:1521/FREEPDB1
----

== Troubleshooting

=== Oracle not starting

Check logs:

[,bash]
----
docker logs oracle-cdc-test
----

The Oracle container can take 1-2 minutes to fully initialize on first start.

=== Connection refused

Ensure Oracle is healthy:

[,bash]
----
docker exec oracle-cdc-test healthcheck.sh
----

=== No CDC events

Verify supplemental logging is enabled:

[,sql]
----
SELECT SUPPLEMENTAL_LOG_DATA_MIN FROM V$DATABASE;
----

Check table-level logging:

[,sql]
----
SELECT LOG_GROUP_NAME, TABLE_NAME
FROM DBA_LOG_GROUPS
WHERE OWNER = 'MY_SCHEMA';
----

== Cleanup

[,bash]
----
docker compose down -v
----

The `-v` flag removes volumes, deleting all database data.
