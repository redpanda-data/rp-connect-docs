= What's New in Redpanda Connect
:description: Summary of new features in Redpanda Connect.
:page-aliases: ROOT:whats_new_rpcn.adoc

This topic includes new content added from version 4.29.0 onwards.

For a full list of product updates, and details of updates prior to version 4.29.0, see the https://github.com/redpanda-data/connect/blob/main/CHANGELOG.md[Redpanda Connect changelog^]. For new Cloud-specific features, see xref:redpanda-cloud:get-started:whats-new-cloud.adoc[].


== Version 4.68.0

link:https://github.com/redpanda-data/connect/releases/tag/v4.68.0[See the full release notes^].

=== Component updates

This release adds the following new components:

* Processors:
** xref:components:processors/jira.adoc[`jira`] (experimental): Executes Jira API queries based on input messages and returns structured results. The processor handles pagination, retries, and field expansion automatically.


This release adds support for the following new fields:

* Inputs components
** xref:components:inputs/redpanda.adoc[`redpanda`]:
*** xref:components:inputs/redpanda.adoc#unordered_processing[`unordered_processing`] - Configures partition consumers to allow parallel and therefore unordered processing of messages of any given partition. This allows for better utilization of processing threads and asynchronous publishing at the output level. The maximum parallelization of each partition is determined by the checkpoint_limit field.

=== Deprecations

The `kafka`, `kafka_franz`, and `redpanda_common` inputs and outputs are now deprecated. Their functionality has been consolidated into the unified `redpanda` input and output components. Migrate existing configurations to use the new `redpanda` components for continued support and access to the latest features.

=== Component updates

This release adds the following new components:

** xref:components:inputs/microsoft_sql_server_cdc.adoc[`microsoft_sql_server_cdc`]: Streams change data from a Microsoft SQL Server database into Redpanda Connect using Change Data Capture (CDC).

This release adds support for the following new fields:

* Inputs components
** xref:components:inputs/redpanda_migrator.adoc[`redpanda_migrator`]
*** xref:components:inputs/redpanda_migrator.adoc#schema_registry[`schema_registry`] - Configuration for schema registry integration. Enables migration of schema subjects, versions, and compatibility settings between clusters.
* Outputs components
** xref:components:outputs/redpanda_migrator.adoc[`redpanda_migrator`]
*** xref:components:outputs/redpanda_migrator.adoc#schema_registry[`schema_registry`] - Configuration for schema registry integration. Enables migration of schema subjects, versions, and compatibility settings between clusters.
*** xref:components:outputs/redpanda_migrator.adoc#consumer_groups[`consumer_groups`] - Whether consumer group offset migration is enabled. When disabled, no consumer group operations are performed.
*** xref:components:outputs/redpanda_migrator.adoc#topic_replication_factor[`topic_replication_factor`] - The replication factor for created topics. If not specified, inherits the replication factor from source topics. Useful when migrating to clusters with different sizes.
*** xref:components:outputs/redpanda_migrator.adoc#sync_topic_acls[`sync_topic_acls`] - Whether to synchronize topic ACLs from source to destination cluster. ACLs are transformed safely: ALLOW WRITE permissions are excluded, and ALLOW ALL is downgraded to ALLOW READ to prevent conflicts.
*** xref:components:outputs/redpanda_migrator.adoc#serverless[`serverless`] - Enable serverless mode for Redpanda Cloud serverless clusters. This restricts topic configurations and schema features to those supported by Redpanda Serverless environments.

=== Deprecations

This release deprecates the individual migrator components and the bundle wrapper in favor of the unified `redpanda_migrator` architecture.

**Deprecated components:**

- `redpanda_migrator` input and output (renamed to `legacy_redpanda_migrator`)
- `redpanda_migrator_offsets` input and output (renamed to `legacy_redpanda_migrator_offsets`)
- `redpanda_migrator_bundle` input and output (now deprecated; these are part of the legacy migration architecture and internally depend on the deprecated `legacy_redpanda_migrator` and `legacy_redpanda_migrator_offsets` components)

These components have been renamed from their previous versions (without the `legacy_` prefix) to preserve backward compatibility. The new `redpanda_migrator` components introduced in this release contain breaking changes. Existing configurations using the previous `redpanda_migrator` components should use these `legacy_` prefixed versions to maintain current functionality.

Users should migrate from deprecated components to the new unified `redpanda_migrator` input/output pair. For detailed migration instructions, see xref:guides:migration/migrate-unified-redpanda-migrator.adoc[].

== Version 4.66.0

=== Component updates

This release adds the following new components:

* Outputs:
** xref:components:outputs/cyborgdb.adoc[`cyborgdb`] (experimental): This output allows you to write vectors to a CyborgDB encrypted index. CyborgDB provides
end-to-end encrypted vector storage with automatic dimension detection and index optimization.

All vector data is encrypted client-side before being sent to the server, ensuring complete
data privacy. The encryption key never leaves your infrastructure.

* Bloblang-functions:
** xref:guides:bloblang/functions.adoc#with_schema_registry_header[`with_schema_registry_header`] (beta): Prepends a 5-byte Schema Registry header to a message. The header consists of a magic byte (0x00) followed by a 4-byte big-endian schema ID.

This release adds support for the following new fields:

* Inputs components
** xref:components:inputs/cassandra.adoc[`cassandra`]
*** xref:components:inputs/cassandra.adoc#reconnect_interval[`reconnect_interval`] - Attempts to reconnect known DOWN nodes in every ReconnectInterval.
*** xref:components:inputs/cassandra.adoc#exponential_reconnection[`exponential_reconnection`] - Optional exponential reconnection policy, this replaces the default constant policy of the driver.
* Outputs components
** xref:components:outputs/cassandra.adoc[`cassandra`]
*** xref:components:outputs/cassandra.adoc#reconnect_interval[`reconnect_interval`] - Attempts to reconnect known DOWN nodes in every ReconnectInterval.

=== Component updates

This release adds:

* Custom TLS fields for the xref:components:inputs/splunk.adoc[`splunk` input] and xref:components:outputs/splunk_hec.adoc[`splunk_hec` output].
* The `timestamp` field to xref:components:outputs/kafka.adoc[`kafka`] and xref:components:outputs/kafka_franz.adoc[`kafka_franz`] outputs.
* The `status_topic` field to the xref:components:redpanda/about.adoc[`redpanda` component].

== Version 4.30.0 to 4.30.1

Redpanda Connect now includes a beta version of the xref:components:inputs/splunk.adoc[`splunk` input].

== Version 4.29.0

Redpanda Connect now separates packages into an enterprise bundle and a free bundle, which contains community and selected certified connectors. To learn more about licensing and support levels, see xref:components:about.adoc#support-levels[].
